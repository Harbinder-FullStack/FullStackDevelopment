Express.js makes web development easy in Node.js, but before diving into it, students should understand the foundational concepts below.

1. JavaScript Fundamentals (ES6+)

  • Variables: let, const, var
  • Functions: normal, arrow, callbacks
  • Objects & Arrays
  • Classes & Inheritance
  • Modules: require/module.exports, import/export
  • Async handling: Promises & async/await

2. Node.js Basics

  • What is Node.js (JavaScript runtime, not a language)
  • Running files: node file.js
  • Core Modules: fs, path, http
  • npm / npx usage
  • package.json basics
  • Using nodemon for auto-reload

3. HTTP Fundamentals

  • Request/Response cycle
  • HTTP Methods: GET, POST, PUT, DELETE
  • Status Codes: 200, 201, 400, 404, 500
  • Headers, Query Params, Path Params
  • REST API basics

4. Manual HTTP Server in Node.js

Example without Express (so students see why Express is useful):

import http from 'http';

const server = http.createServer((req, res) => {
  if (req.url === '/' && req.method === 'GET') {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Hello World');
  } else {
    res.writeHead(404);
    res.end('Not Found');
  }
});

server.listen(3000, () => console.log('Server running on http://localhost:3000'));

5. Middleware Concept

Middleware = a function that sits between request and response, like a security guard checking visitors before entry.

6. JSON & Data Handling

  • Working with JSON.parse() and JSON.stringify()
  • Sending JSON from Node.js
  • Why APIs usually return JSON

7. Async I/O

Understand non-blocking I/O: 

// Non-blocking
fs.readFile('data.txt', 'utf8', (err, data) => {
  console.log(data);
});
console.log('This runs first!');

8. Project Setup

  • Initialize project: npm init -y
  • Organize folders: routes/, controllers/, models/
  • Use environment variables (dotenv)
